# SHA-256 (Fonction de Hachage)

**Tags :** #cryptographie #hachage #sécurité

---

## Définition

**Secure Hash Algorithm 256 bits** — créé par la NSA au début des années 2000.

Une **fonction de hachage** transforme n'importe quelle entrée en une sortie de taille fixe (256 bits / 64 caractères hex).

## Propriétés essentielles

| Propriété | Description |
|---|---|
| **Asymétrique** | Sens unique : impossible de retrouver l'entrée depuis le hash |
| **Déterministe** | Même entrée → toujours même sortie |
| **Taille fixe** | Sortie toujours 256 bits, quelle que soit la taille de l'entrée |
| **Effet avalanche** | 1 bit modifié → sortie totalement différente |
| **Résistance aux collisions** | Infaisable de trouver 2 entrées avec même hash |

## Exemple (effet avalanche)

```
SHA-256("je suis bitcoin")  → 2d71550cf737f4316290401e4f75fd6b...
SHA-256("je suis bitcoin.") → 62e28e5a21fcf90f82fd4a8ae648c036...
                                ↑ totalement différent avec juste un "."
```

## Utilisations dans la blockchain

1. **Adresse Ethereum :** clé publique → SHA-256 → RIPEMD-160 → adresse
2. **Mnemonic Seed :** checksum de la clé privée (8 premiers bits du hash)
3. **Preuve de travail (POW Bitcoin) :** trouver un hash inférieur à une cible
4. **Merkle Tree :** hachage des transactions dans un bloc
5. **Identifiant de bloc :** chaque bloc est identifié par son hash

## RIPEMD-160

Utilisé après SHA-256 pour générer les adresses (réduction de 256 → 160 bits).

```
Clé publique → SHA-256 → RIPEMD-160 → Adresse (40 hex, 20 bytes)
```
