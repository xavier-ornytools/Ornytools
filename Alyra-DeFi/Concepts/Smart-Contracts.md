# 📜 Smart Contracts
Tags: #concept #blockchain #ethereum

---

## Définition
Un **smart contract** est un programme autonome stocké sur la blockchain qui s'exécute automatiquement quand des conditions prédéfinies sont remplies.

> "Code is law" — le contrat s'exécute sans intermédiaire, sans confiance requise.

---

## Analogie
Comme un distributeur automatique :
- Tu insères de l'argent + tu appuies sur un bouton
- La machine exécute automatiquement → te donne le produit
- Pas besoin de faire confiance à un vendeur humain

---

## Propriétés

| Propriété | Explication |
|-----------|-------------|
| **Autonome** | S'exécute seul, personne ne peut l'arrêter |
| **Transparent** | Code lisible par tous sur la blockchain |
| **Immuable** | Une fois déployé, ne peut pas être modifié |
| **Sans confiance** | Trustless — pas besoin de faire confiance à une contrepartie |

---

## Cas d'usage

- **DeFi** → prêts, échanges, liquidité automatisés
- **NFT** → propriété et transfert de tokens uniques
- **DAO** → gouvernance décentralisée par vote
- **ENS** → noms de domaine décentralisés (`.eth`)

---

## Langages
- **Solidity** → langage principal pour Ethereum (similaire JavaScript)
- **Vyper** → alternative plus simple et sécurisée
- **Rust** → utilisé sur Solana

---

## Risques
> Le code est immuable → un bug est permanent

- Hack du DAO (2016) → 60M$ volés, hard fork Ethereum
- Importance de l'audit de code avant déploiement
